### 代码提交规范
[commitlint使用文档](https://github.com/conventional-changelog/commitlint#getting-started)

| 代码提交规范 | 代码提交规范 |
| --- | --- |
|feat：|新功能（feature）|
| fix：| 修补bug |
| docs：| 文档（documentation）|
| style：| 格式（不影响代码运行的变动）|
| refactor：| 重构（即不是新增功能，也不是修改bug的代码变动）|
| test：| 增加测试 |
| chore：| 构建过程或辅助工具的变动 |
| ci: | 构建自动化部署
